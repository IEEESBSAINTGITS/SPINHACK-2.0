## 📌 About SpinHack 2.0
 
**SpinHack 2.0** is an intercollegiate online web development hackathon organized by the IEEE Student Branch, Saintgits College of Engineering. Building on the success of SpinHack 1.0, this edition challenges participants to harness the power of Generative AI to design and develop fully functional websites — all within a high-energy **4-hour sprint**.
 
What makes SpinHack unique? Every team receives their problem statement through a **Spin the Wheel** mechanism at the very start of the event. No pre-preparation, no prior advantage — just raw creativity, technical skill, and the ability to think on your feet. It is a true test of how well a team can adapt, collaborate, and deliver under pressure.
 
Whether you are a frontend enthusiast, a UI/UX thinker, or an AI explorer — SpinHack 2.0 is your platform to showcase what you are made of.

## 🛠️ Technology Stack
 
Participants are expected to use the following tools and technologies:
 
- **Core Tool:** Generative AI platforms — ChatGPT, Gemini, GitHub Copilot, or any equivalent tool
- **Refinement Stack:** HTML, CSS, JavaScript — to enhance and polish AI-generated outputs
- **Version Control & Submission:** GitHub — all development and final submission must be done via GitHub
- **Libraries & APIs:** Open-source libraries and third-party APIs are permitted with proper credits and citations
> 💡 The use of at least **3 different Generative AI tools** must be documented in the project abstract.
 
---
 
## 🏗️ Event Structure
 
The hackathon follows a structured 4-hour flow divided into three key phases:
 
### Phase 1 — Planning
- Teams spin the wheel to receive their assigned problem statement.
- Discuss, strategize, and plan your approach before writing a single line of code.
### Phase 2 — Development
- Build your AI-integrated website using the permitted tools and technologies.
- Make regular commits to your GitHub repository throughout this phase.
- Mentors and technical support will be available — but all development must be done independently.
### Phase 3 — Submission
- Finalize and push all project files to the designated private GitHub repository.
- Ensure the abstract and PPT are included before the deadline.
- No late submissions will be accepted under any circumstances.
---
 
## 📤 Submission Guidelines
 
All teams must submit their work through their designated **private GitHub repository** provided by the organizers. Only team members and the organizing team will have access to each repository.
 
The repository must include the following:
 
- ✅ **Abstract** *(Maximum 1 page)* — A concise document summarizing the project, the approach taken, key findings, and the Generative AI tools used *(minimum 3 tools must be mentioned)*
- ✅ **PowerPoint Presentation (PPT)** — A well-structured presentation clearly explaining the problem statement, proposed solution, and implementation details
- ✅ **All Source Files** — Complete project code with meaningful and regular commits made throughout the event duration
> ⚠️ **Important:** Late submissions will **not** be considered for evaluation under any circumstances. Ensure all files are committed and the repository is complete before the deadline.

## ⚖️ Judging Criteria
 
All submitted projects will be thoroughly evaluated by a **panel of faculty judges** appointed by the organizing committee. Evaluation will be based on the following parameters:
 
| # | Criteria                    | Description                                                              |
|---|-----------------------------|--------------------------------------------------------------------------|
| 1 | Creativity & Innovation     | Originality of the idea and uniqueness of the approach                   |
| 2 | Design & Aesthetics         | Visual appeal, UI consistency, and overall user interface quality         |
| 3 | Functionality & Usability   | How effectively the website works and the quality of user experience      |
| 4 | Use of Generative AI Tools  | Depth, variety, and meaningfulness of AI tool integration                |
| 5 | Relevance to Theme          | How well the solution addresses the assigned problem statement            |
| 6 | Technical Execution         | Code quality, structure, efficiency, and overall technical implementation |
| 7 | Presentation & Pitch        | Clarity, structure, and effectiveness of the PPT                         |
| ⭐ | Bonus                      | Exceptional, creative, or outstanding use of AI beyond expectations      |

## 🔒 Faculty Judges — Final Authority
 
SpinHack 2.0 is evaluated by a distinguished panel of **faculty judges** from Saintgits College of Engineering. Their assessment is based on thorough review of each submission across all judging parameters.

## 📋 Rules & Code of Conduct
 
Participants are required to strictly adhere to the following rules throughout the event:
 
**Originality & Integrity**
- All websites must be original and developed entirely during the hackathon timeframe.
- Plagiarism, copying of existing projects, or submission of pre-built work is strictly prohibited and will result in **immediate disqualification**.
**Development Standards**
- All code must be maintained in the provided GitHub repository with **regular and meaningful commits** throughout the event.
- Teams may seek guidance from available mentors and technical support; however, all stages of development must be executed **independently by the team**.
- Use of open-source libraries and APIs is permitted, provided **proper credits and citations** are included in the project.
**Conduct & Ethics**
- All participants must adhere to the **IEEE Code of Ethics** and the college's code of conduct at all times.
- Any form of misconduct, dishonesty, or disrespect towards fellow participants, mentors, or organizers will result in **instant disqualification**.
- Organizers reserve the right to modify the event schedule, rules, or structure at any time without prior notice.
