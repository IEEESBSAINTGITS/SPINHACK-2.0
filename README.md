# 🌀 SpinHack 2.0
### Intercollegiate Web Development Hackathon
**Organized by IEEE Student Branch | Saintgits College of Engineering**

---

## About

SpinHack 2.0 is an intercollegiate online hackathon where teams build fully functional websites in just **4 hours**. At the start of the event, every team spins a wheel to receive their problem statement — no pre-preparation, no prior advantage. Just creativity, technical skill, and the ability to think fast.

## Technology Stack

- **Core:** Generative AI tools — ChatGPT, Gemini, GitHub Copilot, or any equivalent
- **Refinement:** HTML, CSS, JavaScript — to enhance AI-generated outputs
- **Version Control & Submission:** GitHub

> **Note:** At least **3 different Generative AI tools** must be documented in the project abstract.

Open-source libraries and third-party APIs are permitted with proper credits and citations.

---

## Event Structure

### Phase 1 — Planning
- Spin the wheel to receive your problem statement
- Discuss, strategize, and plan before writing any code

### Phase 2 — Development
- Build your AI-integrated website using the permitted tools
- Make regular, meaningful commits to your GitHub repository throughout this phase
- Mentors and technical support are available, but all development must be done independently

### Phase 3 — Submission
- Finalize and push all project files to your designated private GitHub repository
- Ensure the abstract and PPT are included before the deadline
- **No late submissions will be accepted under any circumstances**

---

## Submission Requirements

All work must be submitted via the **private GitHub repository** provided to each team. The repository must include:

- ✅ **Abstract** *(Max 1 page)* — Summarizing the project, approach, key findings, and the Generative AI tools used *(minimum 3 tools)*
- ✅ **PowerPoint Presentation** — Clearly explaining the problem statement, solution, and implementation
- ✅ **All Source Code** — Complete project files with regular, meaningful commits throughout the event

---

## Judging Criteria

| # | Criteria | Description |
|---|---|---|
| 1 | Creativity & Innovation | Originality of the idea and uniqueness of the approach |
| 2 | Design & Aesthetics | Visual appeal, UI consistency, and overall interface quality |
| 3 | Functionality & Usability | How effectively the website works and the quality of user experience |
| 4 | Use of Generative AI Tools | Depth, variety, and meaningfulness of AI tool integration |
| 5 | Relevance to Theme | How well the solution addresses the assigned problem statement |
| 6 | Technical Execution | Code quality, structure, and overall implementation |
| 7 | Presentation & Pitch | Clarity, structure, and effectiveness of the PPT |
| ⭐ | Bonus | Exceptional or creative use of AI beyond expectations |

> The decisions of the faculty judges and the organizing committee are **final and binding**. No appeals or re-evaluations will be entertained.

## Rules & Code of Conduct

**Originality**
- All websites must be original and developed entirely during the hackathon
- Plagiarism or submission of pre-built work will result in **immediate disqualification**

**Development**
- All code must be maintained in the provided GitHub repository with regular, meaningful commits
- Teams may consult mentors for guidance, but all development must be executed independently
- Open-source libraries and APIs are permitted with proper credits and citations

**Conduct**
- All participants must adhere to the **IEEE Code of Ethics** and the college's code of conduct
- Misconduct, dishonesty, or disrespect towards participants, mentors, or organizers will result in **instant disqualification**
- Organizers reserve the right to modify the event schedule or rules at any time
